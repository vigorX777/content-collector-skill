# Content Collector - 链接内容自动收藏技能

> 自动收集社交媒体精彩内容并整理成结构化笔记存入飞书多维表格

## 背景

在日常上网过程中，我们经常会遇到有价值的内容，如 Twitter 推文、技术博客、微信公众号文章等。这些内容散落在各个平台，难以统一管理和检索。

Content Collector 就是为了解决这个痛点而设计的——自动将用户发送的链接内容收藏到飞书多维表格，形成结构化的知识库。

## 功能特性

### 核心功能

| 功能 | 说明 |
|------|------|
| 自动链接检测 | 发送链接即自动执行，无需额外触发词 |
| 智能分类 | 根据内容关键词自动分类（技术教程/产品想法/工具推荐等）|
| 去重检测 | 自动检测重复链接，避免重复收藏 |
| 批量收藏 | 支持一次发送多个链接批量处理 |
| 图片存储 | 自动保存内容中的图片到飞书素材库 |
| 多平台支持 | 支持 X/Twitter、微信公众号、飞书、小红书、抖音、知乎、掘金等 |
| 长文本处理 | 自动分页获取超长内容 |

### 触发方式

- **显式触发**：发送链接 + 关键词（收藏、收录、推荐、mark等）
- **自动检测**：发送链接即可自动执行（无需任何关键词）

## 前提准备

### 1. OpenClaw 环境

确保已安装 OpenClaw 并配置好飞书插件。

### 2. 飞书 OAuth 授权（重要！）

**为什么需要授权？**
默认情况下，内容以 Bot 应用身份创建，用户无法编辑。完成授权后，内容将以用户身份创建，用户拥有完全控制权。

**授权步骤：**

1. **配置 OpenClaw**
   在 `openclaw.json` 的 `channels.feishu` 中添加 `perm: true`：
   ```json
   "channels": {
     "feishu": {
       "enabled": true,
       "perm": true,
       "appId": "your_app_id",
       "appSecret": "your_app_secret"
     }
   }
   ```

2. **触发授权**
   发送任意链接并说"收藏"，例如：
   > "https://x.com/i/status/xxx 收藏"

3. **完成授权**
   飞书会自动发送授权卡片，点击卡片上的链接完成授权。

### 3. 目标多维表格

需要创建一个飞书多维表格用于存储收藏内容，表格需要包含以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 标题 | 文本 | 内容标题 |
| 分类 | 单选 | 🔧工具推荐/📖技术教程/🛠️实战案例/💡产品想法/🔥热点资讯/📄其他 |
| 来源 | 文本 | 如 X/Twitter @username |
| 原文链接 | 超链接 | 原始 URL |
| 原文内容 | 文本 | 抓取的完整内容 |
| 记录时间 | 创建时间 | 自动记录 |

## 使用方式

### 方式一：发送链接 + 触发词

```
用户：https://x.com/i/status/xxx 收藏
Bot：✅ 已收藏到飞书多维表格
```

支持的触发词：
- 收藏、收录、转存、保存、存档
- 推荐、mark、记一下
- 转发、整理

### 方式二：仅发送链接（自动检测）

```
用户：https://x.com/i/status/xxx
Bot：✅ 已自动收藏
```

### 方式三：批量收藏

```
用户：
https://x.com/1 收藏
https://x.com/2 收藏
https://x.com/3 收藏

Bot：✅ 已批量收藏 3 条内容
```
